Babbitt Lab at RIT

Our research is focused upon gaining a better understanding of how the biophysical function of proteins, DNA and other molecules constrains molecular evolution. We employ a computational approach, taking advantage of many methods in computer simulation, machine learning, statistics and traditional bioinformatics. Our research group consists of primarily undergraduate students working in many programs across RIT. Most students work for the lab on a volunteer basis, with very flexible schedules, working towards publishable projects that can be added to a students' resume. Students interested in working with the lab should contact me at gabsbi(at)rit.edu.

Comparing Protein Dynamics

machine learning assisted visualization of comparative protein dynamics (DROIDS/maxDemon v4.0) Website/GitHub

DROIDS is a machine learning assisted GUI based AMBER or OpenMM MD pipeline for statistical comparison of protein dynamics (python DROIDS.py). The DROIDS program runs multiple replicate MD runs in two comparative states (e.g. bound vs unbound, wild-type vs mutated, or hot vs cold) and statistically compares and color-maps differences in local atom fluctuations across the protein. The subsequent maxDemon program trains a multi-agent machine learning algorithm upon the dynamics in these two comparative states, identifying regions of functionally or evolutionary conserved dynamics, and exploring impacts upon dynamics in genetic and drug class variants.

Generating Protein Structural Variants

convert multiple sequence alignment to variant PDB structures (PDBmutator v1.4) Website/GitHub

PDBmutator is a GUI enabled script to batch run swapna, swapaa and delete residue commands in UCSF Chimera on a PDB file. Users can generate all possible structural variants OR variants described by a multiple sequence alignment file. It requires pdb4amber (from AmberTools 18 https://ambermd.org/) or directly from GitHub (https://github.com/Amber-MD/pdb4amber) if an alignment file is used as the source of the PDB variants. All variants should be subjected to equilibration via molecular dynamics after they are generated.

Fast+Easy Molecular Dynamics

A simple graphical user interface for GPU accelerated molecular dynamics via Amber/AmberTools or OpenMM/AmberTools (amberMDgui v1.3) Website/GitHub

A simple graphical user interface for running ensembles of GPU accelerated AMBER molecular dynamics simulations, and viewing them in UCSF Chimera. If users do not have a license for GPU accelerated AMBER (i.e. pmemd.cuda), users can select the OpenMM molecular dynamics engine as an option for GPU accelerated molecular dynamics.

Exploring Drug+Protein Binding Energy

Calculate ligand binding enthalpy from water balanced molecular dynamic simulations via Amber or OpenMM (amberENTHALPY v3.0) Website/GitHub

A simple graphical user interface for running small molecule ligand-protein binding enthalpy calculations from GPU accelerated molecular dynamics simulations, and viewing them in UCSF Chimera. The method sets up replicate MD simulations in 4 states (protein-ligand complex, protein host alone, ligand guest alone and water alone) following the method of Fenley and Gilson (2014) published here https://pubs.acs.org/doi/abs/10.1021/ct5004109. This computer simulation approach is intended to mimic the results obtained through Isothermal Titration Calorimetry.

Tracking Evolving Protein Stability

Compute free energy impacts of mutation (via MAESTRO) on a phylogenetic tree (evolMAESTRO -not yet fully released) Website/GitHub

This is an ongoing project GUI enabled pipeline) to compute and visualize delta delta G on a phylogenetic tree

Our software is currently tested on Linux Mint 18 Serena/Sylvia and Linux Mint 19 Tara. Our typical Linux builds include 1 or 2 high end Nvidia graphics cards. The dual GPU options in our GUI's do not assume SLI connections between cards.

Main software dependencies include python kivy, perl tk, Amber/AmberTools, UCSF Chimera/ChimeraX, MAESTRO, OpenMM and the R project for statistical computing.
